## Playbook Overview

The **Sustaining Engineering Playbook** is a comprehensive guide for engineers working on extended lifecycle support for Red Hat products. It covers:

- **Team Structure** - Organization, leadership, and contact information
- **Communication Channels** - Mailing lists, Slack channels, and calendars
- **Onboarding** - Checklist and setup guides for new team members
- **Product Areas** - RHEL, OpenShift, OpenJDK, Satellite, and RHOAI sustaining
- **Tools & Infrastructure** - Jenkins, Konflux, Errata Tool, Brew, ART tools
- **Workflows & Processes** - Jira ticket processing, backporting workflow, z-stream releases
- **Release Process** - Detailed OCP z-stream release steps (Konflux pipeline)
- **Quality Engineering** - Testing tools, environments, CI/CD, and AI-assisted testing
- **Troubleshooting** - Common release errors and solutions
